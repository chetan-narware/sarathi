Sarthi Web App - Backend Information

Overview

The Sarthi Web App is designed to simulate autonomous vehicle functionalities, specifically lane detection and pothole detection. The backend is built using Flask and handles various routes for uploading videos, processing them, and providing damage reports.

Backend Routes

  • /upload/lane (POST) - Upload a video for lane detection. The video is processed, and lane markings are detected.
  • /upload/Damage (POST) - Upload a video for pothole detection. The video is processed to detect potholes.
  • /stream/lane/uploads (GET) - Stream the lane detection video after processing.
  • /stream/Damage/uploads (GET) - Stream the pothole detection video after processing.
  • /api/upload-image (POST) - Upload an image to report damage. The image is stored with geolocation information.
  • /api/get-markers (GET) - Fetch a list of damage markers (geolocation data) from the database.

How It Works

The app utilizes various techniques like lane detection using OpenCV and pothole detection using YOLOv8. Users can upload videos for processing, and the results are returned via video streaming. The app also allows users to upload images with their geolocation data to report damage on the road.

Data Storage

The app stores geolocation data of reported damage using a database. This data can be accessed through the /api/get-markers route.

Technologies Used

  • Flask (for backend API)
  • OpenCV (for lane detection)
  • YOLOv8 (for pothole detection)
  • Firebase (for storing images)
  • HTML/CSS/JavaScript (for frontend)
